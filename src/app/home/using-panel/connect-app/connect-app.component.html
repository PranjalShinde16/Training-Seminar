Set up My App?

After logging into your account's dashboard, select “Applications” and then “Create New”.

OTPless works with iOS and Android operating systems and all web browsers.

Before setting up your app, you need to determine which platforms you will be using. Then you can set up your application on these platforms individually.

Android

If you’d like to use the OTPless verification solution in your Android app, select the “Android” as Application Type.

Enter your Application Name. This Name will be shown when a User receives the link on his or her whatsapp.

After that You’ll need to enter a callback URI to redirect your users to the relevant pages in your app and type in the Package Name you got from Google Play Store.

callback URI is necessary for your users to return to the app after they complete the verification process. You can find more information on callback URI integration here.

Enter your callback URI and Package Name, then click Save. The page will refresh automatically and your Client Id, and Client Secret will be displayed on the screen.

For more information on how to use the Client Id, and the Client Secret, click here.

iOS

If you’d like to use OTPless in your iOS app, select the “iOS” as Application Type.

Enter your Application Name. This Name will be shown when a User receives the link on his or her whatsapp.

After that You’ll need to enter a callback URI to redirect your users to the relevant pages in your app, and type in the App Store Bundle ID.

callback URI is necessary for your users to return to the app after they complete the verification process. You can find more information on callback URI integration here.

Enter your callback URI and Bundle ID, then click Save. The page will refresh automatically and your Client Id, and Client Secret will be displayed on the screen.

For more information on how to use the Client Id, and the Client Secret, click here.

Web SDK

If you’d like to use the OTPless verification solution in your Web app, select the “Web Application” as Application Type.

Enter your Application Name. This Name will be shown when a User receives the link on his or her whatsapp.

After that You’ll need to enter a full callback URI to redirect your users to the relevant pages in your website.

callback URI is necessary for your users to return to the website after they complete the verification process. You can find more information on callback URI integration here.

For more details on integration, click here.

Rest API

The Rest API platform will automatically be defined for all your apps. You can also integrate OTPless using Rest API. For more details on how to integrate it into your app using the server-key parameter, click here.

Need to Know

What Are callback URI?

callback URI allow users who complete the verification process with WhatsApp to return to your application. It’s crucial that your callback URI settings are done properly in order to ensure a successful verification process. Callback URI can be either an https URL or a custom URI scheme for your android/iOS App.

What Is a Client Id and a Client Secret?

The Client Id and Client Secret parameters are used in all REST API requests that are sent to OTPless:

You’ll need to send this parameter to OTPless as a header parameter.

IMPORTANT:

The End-user should never have access to the Client Id and Client Secret parameters. Never hardcode this parameter into the codebase of you iOS and Android app and do not share it with anyone. If you lose this parameter, retrieve it from your app details page in the panel or contact us as soon as possible at support@otpless.com
